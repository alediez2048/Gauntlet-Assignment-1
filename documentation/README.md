# Documentation

Project documentation is grouped by category:

| Directory | Purpose |
|-----------|---------|
| **architecture/** | System design, data flow, state ownership, event schemas |
| **requirements/** | Product requirements (PRD), ticket breakdown, acceptance criteria |
| **testing/** | Testing strategy, per-ticket checklists, E2E and Vitest guides |
| **agents/** | AI agent context (agents.md), quick reference (CLAUDE.md) |
| **reference/** | Pre-search notes, recommended file structure |
| **tickets/** | Progress tracker (TICKETS.md), dev log (DEV-LOG.md), ticket primers (TICKET-02 through TICKET-04, summaries) |
